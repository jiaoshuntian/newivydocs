# pgsql-http

pgsql-http is an open-source extension that lets you initiate HTTP requests directly within IvorySQL — acting as a built-in web client. It bridges the gap between the database and external web services, enabling interaction with APIs through simple SQL function calls.

With this extension, you can:

- Retrieve network data (`GET`), submit data (`POST`/`PUT`), update (`PATCH`), or delete (`DELETE`) remote resources in SQL queries, triggers, or stored procedures.
- Set request headers, perform URL encoding, send JSON data, and parse response status, headers, and content.

## Typical Use Cases

- **Real-time data retrieval** — fetch exchange rates, weather data, or other external APIs and store results directly in tables.
- **Trigger-based notifications** — automatically notify microservices when data changes.
- **ETL pipelines** — clean data in the database and submit it directly to external APIs.

## Installation

The pgsql-http plugin is already included in the IvorySQL installation package. If you installed IvorySQL via the official package, no manual installation is needed.

### Source Installation

<Callout type="info">
These steps assume Ubuntu 24.04 (x86_64) with IvorySQL v5+ installed at `/usr/local/ivorysql/ivorysql-5`.
</Callout>

<Steps>

### Step 1: Install dependencies

pgsql-http depends on `libcurl` development files:

```bash
sudo apt install libcurl4-openssl-dev
```

### Step 2: Download and build

Download the [1.7.0 source package](https://github.com/pramsey/pgsql-http/releases/tag/v1.7.0):

```bash
tar xvf pgsql-http-1.7.0.tar.gz
cd pgsql-http-1.7.0
# Ensure pg_config is accessible in PATH
make
sudo make install
```

</Steps>

## Enable and Verify

```sql
CREATE EXTENSION http;

SELECT * FROM pg_available_extensions WHERE name = 'http';
```

Expected output:

```
   name    | default_version | installed_version |       comment
-----------+-----------------+-------------------+-------------------------------------------------------------------------
 http      | 1.7             | 1.7               | HTTP client for PostgreSQL, allows web page retrieval inside the database.
(1 row)
```

## Usage

### GET Request

Fetch data from an external URL and access the response:

```sql
SELECT content
FROM http_get('https://httpbin.org/get');
```

The response is returned as a `http_response` composite type with fields: `status`, `content_type`, `headers`, and `content`.

```sql
SELECT status, content_type, content::json
FROM http_get('https://httpbin.org/get');
```

### POST Request with JSON

Send a JSON payload to an external API:

```sql
SELECT status, content
FROM http_post(
    'https://httpbin.org/post',
    '{"key": "value"}',
    'application/json'
);
```

### Setting Custom Headers

Use `http_header` to attach custom headers (for example, an Authorization token):

```sql
SELECT status, content
FROM http((
    'GET',
    'https://httpbin.org/bearer',
    ARRAY[http_header('Authorization', 'Bearer my-token')],
    NULL,
    NULL
)::http_request);
```

### Store External Data in a Table

Retrieve JSON from an external API and insert selected fields into a table:

```sql
CREATE TABLE exchange_rates (
    fetched_at timestamptz DEFAULT now(),
    rate       numeric
);

INSERT INTO exchange_rates (rate)
SELECT (content::json->>'rate')::numeric
FROM http_get('https://api.example.com/rates/usd');
```

<Callout type="info">
pgsql-http runs HTTP requests synchronously inside the transaction. For high-frequency use, consider calling it from a background worker or via pg_cron to avoid blocking client connections.
</Callout>
