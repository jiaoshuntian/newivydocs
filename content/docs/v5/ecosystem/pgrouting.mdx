# pgRouting

pgRouting is an open-source geospatial routing extension library built on PostgreSQL and PostGIS. It endows the database with powerful network analysis capabilities, enabling complex path planning and graph theory computation — such as calculating the shortest path between two points, performing Traveling Salesman Problem (TSP) analysis, or computing service area coverage.

By embedding routing algorithms directly into the database, pgRouting avoids complex data transfer and computation at the application layer. It leverages PostgreSQL's data management capabilities and PostGIS's spatial functions to perform efficient computation on spatial network data directly within the database.

## Use Cases

- Logistics and distribution
- Traffic navigation
- Network analysis
- Urban planning
- Supply chain management

## Installation

<Callout type="info">
These steps assume IvorySQL v5+ is installed at `/usr/local/ivorysql/ivorysql-5`.
</Callout>

### Prerequisites

- Perl (typically already installed with IvorySQL)
- CMake >= 3.12
- Boost >= 1.56

<Steps>

### Step 1: Install dependencies

```bash
sudo apt install cmake libboost-all-dev
```

### Step 2: Download and compile

```bash
wget https://github.com/pgRouting/pgrouting/releases/download/v3.8.0/pgrouting-3.8.0.tar.gz
tar xvf pgrouting-3.8.0.tar.gz
cd pgrouting-3.8.0
mkdir build
cd build
cmake .. -DPOSTGRESQL_PG_CONFIG=/path/to/pg_config  # e.g. /usr/local/ivorysql/ivorysql-5/bin/pg_config
make
sudo make install
```

</Steps>

## Usage

pgRouting works on road network data stored as a table of edges. Each edge needs at minimum a unique ID, a source node, a target node, and a cost.

### Create a Road Network Table

```sql
CREATE TABLE roads (
    id     bigserial PRIMARY KEY,
    source integer,
    target integer,
    cost   float
);

INSERT INTO roads (source, target, cost) VALUES
    (1, 2, 1.0),
    (2, 3, 2.0),
    (3, 4, 1.5),
    (1, 4, 5.0);
```

### Shortest Path (Dijkstra)

Find the shortest path between node 1 and node 4:

```sql
SELECT seq, node, edge, cost, agg_cost
FROM pgr_dijkstra(
    'SELECT id, source, target, cost FROM roads',
    1,    -- start node
    4     -- end node
);
```

Expected output:

```
 seq | node | edge | cost | agg_cost
-----+------+------+------+----------
   1 |    1 |    1 |    1 |        0
   2 |    2 |    2 |    2 |        1
   3 |    3 |    3 |  1.5 |        3
   4 |    4 |   -1 |    0 |      4.5
(4 rows)
```

### Traveling Salesman Problem (TSP)

Find an efficient route visiting a set of nodes:

```sql
SELECT seq, node
FROM pgr_TSP(
    $$SELECT * FROM pgr_dijkstraCostMatrix(
        'SELECT id, source, target, cost FROM roads',
        ARRAY[1, 2, 3, 4]
    )$$
);
```

Most pgRouting functions require PostGIS to be installed for spatial geometry support. Install PostGIS first if you plan to use geographic coordinate systems. See the [PostGIS](/docs/v5/ecosystem/postgis) guide for setup instructions.
</Callout>
