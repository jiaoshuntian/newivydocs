# Anonymous Block

Anonymous blocks let you write and execute PL/iSQL procedural code on the fly, without storing it as a named function or procedure in the database catalog. This is the Oracle-compatible equivalent of PL/SQL anonymous blocks.

## How It Works

An anonymous block is parsed and executed immediately by the server. Unlike a stored function or procedure, it is not persisted in the system directory. IvorySQL is compatible with Oracle's anonymous block syntax format, with support spanning the client tool (psql), the frontend parser, and the backend executor.

## Syntax

Oracle-style anonymous blocks use the `DECLARE ... BEGIN ... END;` structure, terminated by a forward slash (`/`) on its own line in psql:

```sql
DECLARE
  -- variable declarations
BEGIN
  -- executable statements
END;
/
```

The `DECLARE` section is optional. If there are no variable declarations, you can start directly with `BEGIN`:

```sql
BEGIN
  -- executable statements
END;
/
```

<Callout type="info">
  The forward slash (`/`) on its own line is the Oracle-style statement terminator in psql. It signals the end of the anonymous block and triggers execution.
</Callout>

## Examples

### Simple anonymous block

```sql
DECLARE
  i integer := 10;
BEGIN
  RAISE NOTICE '%', i;
END;
/
-- NOTICE:  10
```

### Block with CASE and exception handling

```sql
DECLARE
  grade CHAR(1);
BEGIN
  grade := 'B';
  CASE grade
    WHEN 'A' THEN RAISE NOTICE 'Excellent';
    WHEN 'B' THEN RAISE NOTICE 'Very Good';
  END CASE;
EXCEPTION
  WHEN CASE_NOT_FOUND THEN
    RAISE NOTICE 'No such grade';
END;
/
-- NOTICE:  Very Good
```

### Block with a loop

```sql
DECLARE
  total integer := 0;
BEGIN
  FOR i IN 1..5 LOOP
    total := total + i;
  END LOOP;
  RAISE NOTICE 'Sum = %', total;
END;
/
-- NOTICE:  Sum = 15
```

### Block that queries data

```sql
DECLARE
  emp_name varchar(100);
BEGIN
  SELECT first_name INTO emp_name FROM employees WHERE employee_id = 1;
  RAISE NOTICE 'Employee: %', emp_name;
EXCEPTION
  WHEN NO_DATA_FOUND THEN
    RAISE NOTICE 'No employee found';
END;
/
```

<Callout type="tip">
  Anonymous blocks support the full PL/iSQL feature set: variables, cursors, loops, conditional logic, exception handling, and calls to stored procedures and functions.
</Callout>
