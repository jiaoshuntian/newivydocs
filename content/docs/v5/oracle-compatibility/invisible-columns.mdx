# Invisible Columns

Invisible columns let you conceal specific columns during operations like `SELECT *`. Columns must be explicitly named in queries to be accessed. This is useful for:

- **Application migration** — Add new columns without disrupting existing applications that rely on `SELECT *`.
- **Data management** — Hide internal or sensitive columns from generic access.

## Create a Table with Invisible Columns

```sql
CREATE TABLE mytable (a INT, b INT INVISIBLE, c INT);
```

## Insert Values

You must explicitly specify invisible columns in the column list:

```sql
-- Correct: column list includes the invisible column
INSERT INTO mytable (a, b, c) VALUES (1, 2, 3);
INSERT 0 1

-- Error: omitting the column list
INSERT INTO mytable VALUES (4, 5, 6);
-- ERROR: INSERT has more expressions than target columns
```

## Query Data

```sql
-- SELECT * does not show invisible columns
SELECT * FROM mytable;
--  a | c
-- ---+---
--  1 | 3

-- Explicitly name the invisible column
SELECT a, b, c FROM mytable;
--  a | b | c
-- ---+---+---
--  1 | 2 | 3
```

## Inspecting in psql

In Oracle mode, `\d+` shows the invisible status:

```sql
\d+ mytable
--                                                    Table "public.mytable"
--  Column |      Type       | Collation | Nullable | Default | Invisible | Storage
-- --------+-----------------+-----------+----------+---------+-----------+---------
--  a      | pg_catalog.int4 |           |          |         |           | plain
--  b      | pg_catalog.int4 |           |          |         | invisible | plain
--  c      | pg_catalog.int4 |           |          |         |           | plain
```

## Limitations

- **ALTER COLUMN** to change visibility is not currently supported; this will be enhanced in a future version.
- **Column ordering** — In Oracle, invisible columns are not included in column ordering. This behavior is not yet addressed in IvorySQL.
