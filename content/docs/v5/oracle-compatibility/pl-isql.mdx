# PL/iSQL & Stored Procedures

IvorySQL provides a PL/iSQL procedural language that is compatible with Oracle's PL/SQL. When Oracle clauses have a direct equivalent in IvorySQL, they are mapped automatically. When they do not, the syntax is implemented even if the underlying behavior differs.

## Function Syntax

### EDITIONABLE / NONEDITIONABLE

```sql
CREATE OR REPLACE EDITIONABLE FUNCTION ora_func RETURN integer AS
BEGIN
  RETURN 1;
END;
/

CREATE OR REPLACE NONEDITIONABLE FUNCTION ora_func RETURN integer IS
BEGIN
  RETURN 1;
END;
/
```

### RETURN and IS keywords

```sql
CREATE OR REPLACE EDITIONABLE FUNCTION ora_func RETURN integer IS
BEGIN
  RETURN 1;
END;
/
```

### Functions without parameters (no parentheses required)

```sql
CREATE OR REPLACE FUNCTION ora_func RETURN integer
SHARING = METADATA
IS
BEGIN
  RETURN 1;
END;
/
```

### No DECLARE keyword before variables

Unlike PL/pgSQL, PL/iSQL does not require `DECLARE` before variable declarations in the function body:

```sql
CREATE OR REPLACE FUNCTION ora_func(num1 IN int, num2 IN int)
RETURN int
AS
  num3 int := 10;
  num4 int := 10;
  num5 int;
BEGIN
  num3 := num1 + num2;
  num4 := num1 * num2;
  num5 := num3 * num4;
  RETURN num5;
END;
/

SELECT ora_func(5, 9) FROM dual;
--  ora_func
-- ----------
--       630
```

### Slash (`/`) as statement terminator

In psql, use `/` to terminate a `CREATE FUNCTION` or `CREATE PROCEDURE` statement:

```sql
CREATE OR REPLACE FUNCTION my_func RETURN integer IS
BEGIN
  RETURN 42;
END;
/
```

### NOCOPY for OUT parameters

```sql
CREATE OR REPLACE FUNCTION test_nocopy(
  a IN int,
  b OUT NOCOPY int,
  c IN OUT NOCOPY int
) RETURN int IS
BEGIN
  b := a;
  c := a;
  RETURN 1;
END;
/
```

### SHARING clause

```sql
CREATE OR REPLACE FUNCTION ora_func RETURN integer
SHARING = METADATA
IS
BEGIN
  RETURN 1;
END;
/

CREATE OR REPLACE FUNCTION ora_func RETURN integer
SHARING = NONE
IS
BEGIN
  RETURN 1;
END;
/
```

### AUTHID (Invoker Rights)

The default permission is `DEFINER`.

```sql
CREATE OR REPLACE FUNCTION ora_func RETURN integer
SHARING = NONE AUTHID CURRENT_USER
IS
BEGIN
  RETURN 1;
END;
/

CREATE OR REPLACE FUNCTION ora_func RETURN integer
SHARING = NONE AUTHID DEFINER
IS
BEGIN
  RETURN 1;
END;
/
```

### ACCESSIBLE BY clause

Restricts which objects can call a function:

```sql
CREATE OR REPLACE FUNCTION ora_func RETURN integer
SHARING = NONE AUTHID DEFINER
ACCESSIBLE BY (FUNCTION A.B, PROCEDURE C.D, PACKAGE E, TRIGGER F, TYPE G)
IS
BEGIN
  RETURN 1;
END;
/
```

### DEFAULT COLLATION

```sql
CREATE OR REPLACE FUNCTION ora_func RETURN integer
SHARING = NONE AUTHID DEFINER
ACCESSIBLE BY (FUNCTION A.B, PROCEDURE C.D)
DEFAULT COLLATION USING_NLS_COMP
IS
BEGIN
  RETURN 1;
END;
/
```

### DETERMINISTIC

```sql
CREATE OR REPLACE FUNCTION ora_func RETURN integer
SHARING = NONE AUTHID DEFINER
DETERMINISTIC
IS
BEGIN
  RETURN 1;
END;
/
```

### PARALLEL_ENABLE

```sql
CREATE OR REPLACE FUNCTION ora_func RETURN integer
SHARING = NONE AUTHID DEFINER
PARALLEL_ENABLE
IS
BEGIN
  RETURN 1;
END;
/
```

### RESULT_CACHE

```sql
CREATE OR REPLACE FUNCTION ora_func RETURN integer
SHARING = NONE AUTHID DEFINER
RESULT_CACHE RELIES_ON (data_source1, data_source2)
IS
BEGIN
  RETURN 1;
END;
/
```

### PIPELINED

```sql
CREATE OR REPLACE FUNCTION ora_func RETURN integer
PIPELINED
IS
BEGIN
  RETURN 1;
END;
/
```

### SQL_MACRO

```sql
CREATE OR REPLACE FUNCTION ora_func RETURN integer
SQL_MACRO
IS
BEGIN
  RETURN 1;
END;
/
```

### ALTER FUNCTION

```sql
ALTER FUNCTION public.test_func NONEDITIONABLE;
ALTER FUNCTION test_func COMPILE;
ALTER FUNCTION test_func COMPILE DEBUG;
ALTER FUNCTION test_func COMPILE DEBUG SD = MV;
ALTER FUNCTION test_func COMPILE DEBUG REUSE SETTINGS;
```

## Procedure Syntax

### EDITIONABLE / NONEDITIONABLE

```sql
CREATE OR REPLACE EDITIONABLE PROCEDURE ora_procedure
IS
  p integer := 20;
BEGIN
  RAISE NOTICE '%', p;
END;
/

CREATE OR REPLACE NONEDITIONABLE PROCEDURE ora_procedure
IS
  p integer := 20;
BEGIN
  RAISE NOTICE '%', p;
END;
/
```

### Procedures without parameters

```sql
CREATE OR REPLACE EDITIONABLE PROCEDURE ora_procedure
IS
  p integer := 20;
BEGIN
  RAISE NOTICE '%', p;
END;
/
```

### SHARING clause

```sql
CREATE OR REPLACE PROCEDURE ora_procedure
SHARING = METADATA
IS
  p integer := 20;
BEGIN
  RAISE NOTICE '%', p;
END;
/
```

### AUTHID

```sql
CREATE OR REPLACE PROCEDURE ora_procedure
SHARING = METADATA
AUTHID CURRENT_USER
IS
  p integer := 20;
BEGIN
  RAISE NOTICE '%', p;
END;
/
```

### ALTER PROCEDURE

```sql
ALTER PROCEDURE test_proc EDITIONABLE;
ALTER PROCEDURE public.test_proc NONEDITIONABLE;
ALTER PROCEDURE test_proc COMPILE;
ALTER PROCEDURE test_proc COMPILE DEBUG;
```

## Calling Functions Without Parentheses

Functions and procedures can be called without parentheses when they have no parameters:

```sql
CREATE OR REPLACE FUNCTION f_noparentheses RETURN int IS
BEGIN
  RETURN 11;
END;
/

SELECT f_noparentheses FROM dual;

CREATE OR REPLACE PROCEDURE protest AS
BEGIN
  RAISE NOTICE 'protest';
END;
/

CALL protest();
```

## Related System Views

Oracle-compatible catalog views are available in `contrib/ivorysql_ora/src/sysview/sysview--1.0.sql`, including:

- `DBA_PROCEDURES`, `ALL_PROCEDURES`, `USER_PROCEDURES`
- `DBA_SOURCE`, `ALL_SOURCE`, `USER_SOURCE`
- `DBA_ARGUMENTS`, `ALL_ARGUMENTS`, `USER_ARGUMENTS`

## Backup

When `pg_dump` exports functions and procedures, it appends a slash (`/`) at the end of each definition to ensure compatibility with Oracle-style terminators.
