# Oracle Compatibility

IvorySQL lets you run Oracle workloads on a fully open-source PostgreSQL foundation — with minimal or zero code changes. Whether you are migrating an existing Oracle application or building new code that targets Oracle syntax, IvorySQL's dual-mode architecture handles the translation transparently.

## How it works

IvorySQL ships with two compatibility layers working together:

- **Dual port** — connect on port 1521 to get Oracle mode automatically. Port 5432 behaves as standard PostgreSQL.
- **Dual parser** — a separate parser handles Oracle-specific syntax so PG and Oracle syntax paths don't interfere with each other.
- **PL/iSQL** — a procedural language compatible with Oracle's PL/SQL, supporting packages, stored procedures, anonymous blocks, and more.
- **GUC parameters** — fine-grained control over compatibility behavior via `ivorysql.compatible_mode` and NLS settings.

<Callout type="tip">
**Starting a migration from Oracle?** Begin with [PL/iSQL & Stored Procedures](/docs/v5/oracle-compatibility/pl-isql) if you have procedural code, or [Data Types & Built-ins](/docs/v5/oracle-compatibility/data-types) if you need to map Oracle types. Use [Migrate to IvorySQL](/docs/v5/migration) to automate schema and data export with Ora2Pg.
</Callout>
