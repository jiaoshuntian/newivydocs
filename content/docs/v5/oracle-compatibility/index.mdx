# Oracle Compatibility

IvorySQL is an advanced PostgreSQL distribution that provides deep Oracle compatibility, enabling you to run Oracle workloads with minimal or zero code changes. Whether you are migrating an existing Oracle database or building a new application that targets Oracle syntax, IvorySQL gives you the tools to do it on a fully open-source, community-driven foundation.

## Dual-Mode Architecture

IvorySQL runs in one of two modes, determined at cluster initialization time:

- **Oracle mode** (default) — the database understands and executes Oracle-compatible SQL syntax, PL/iSQL procedural language, and Oracle built-in functions.
- **PG mode** — standard PostgreSQL behavior, with no Oracle compatibility layer active.

You select the mode when initializing a cluster using the `-m` flag on `initdb`:

```bash
# Oracle-compatible mode (default)
initdb -D ../data

# Explicitly Oracle mode
initdb -D ../data -m oracle

# PostgreSQL mode — no Oracle compatibility
initdb -D ../data -m pg
```

If the `-m` parameter is not specified, the cluster defaults to Oracle-compatible mode. Once a cluster is initialized in PG mode, Oracle syntax is not available and the mode cannot be switched.

## How It Works

IvorySQL's compatibility is implemented across several layers:

- **Dual-port design** — Port 5432 behaves as standard PostgreSQL. Port 1521 (default) automatically enables Oracle compatibility mode for all connections. If you connect via 5432 and need Oracle mode, set `ivorysql.compatible_mode = oracle`.
- **Dual parser** — A separate parser module handles Oracle-specific syntax, keeping PG and Oracle syntax paths isolated to minimize interference.
- **PL/iSQL procedural language** — A new procedural language compatible with Oracle's PL/SQL, supporting anonymous blocks, nested subfunctions, packages, and more.
- **GUC framework** — A unified configuration framework for Oracle compatibility parameters like `ivorysql.compatible_mode`, `nls_date_format`, and others.
