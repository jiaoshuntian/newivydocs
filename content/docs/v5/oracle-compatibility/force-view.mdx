# Force View

Force View lets you create views even when their dependencies (tables, functions) do not yet exist. This is essential during Oracle-to-IvorySQL migrations when view definitions must be deployed before all underlying objects are ready.

## How It Works

- **`CREATE FORCE VIEW`** creates a view placeholder even with missing dependencies, returning `WARNING: View created with compilation errors`.
- **Automatic compilation** — when the view is accessed, IvorySQL attempts to recompile. If dependencies are now present, the view becomes a normal view. If not, it raises `view "<schema>.<name>" has errors`.
- **Dependency fallback** — if a normal view's dependencies are removed, IvorySQL automatically converts it to a Force view, preserving the definition for later recovery.

## Examples

### Create a Force View with Missing Dependencies

```sql
CREATE FORCE VIEW fv_customer AS
  SELECT c_id, c_name FROM missing_customer;
-- WARNING: View created with compilation errors
```

### Automatic Compilation

Once the missing table is created, accessing the view triggers automatic recompilation:

```sql
CREATE TABLE missing_customer(
  c_id   int PRIMARY KEY,
  c_name text
);
INSERT INTO missing_customer VALUES (1, 'Alice');

-- Accessing the view triggers auto-compilation
SELECT * FROM fv_customer;
--  c_id | c_name
-- ------+--------
--     1 | Alice
```

### Explicit Compilation and Failure Fallback

```sql
-- Recreate as Force view with a new missing dependency
CREATE OR REPLACE FORCE VIEW fv_customer AS
  SELECT c_id, upper(c_name) AS c_name FROM missing_customer_v2;

-- Explicit compilation fails
ALTER VIEW fv_customer COMPILE;
-- WARNING: View altered with compilation errors

-- Supply the missing dependency
CREATE TABLE missing_customer_v2(
  c_id   int,
  c_name text
);

-- Compile again — succeeds
ALTER VIEW fv_customer COMPILE;
-- ALTER VIEW
```
