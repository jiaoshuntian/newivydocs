# Tool Reference

This page contains reference documentation for all IvorySQL client applications and server utilities. These tools can be run on any host, independent of where the database server resides.

When specified on the command line, user and database names have their case preserved -- the presence of spaces or special characters might require quoting. Table names and other identifiers do not have their case preserved, except where documented, and might require quoting.

## Overview

### Client Applications

| Tool | Description |
|------|-------------|
| `clusterdb` | Recluster tables previously marked as clustered. |
| `createdb` | Create a new IvorySQL database. |
| `createuser` | Create a new IvorySQL role. |
| `dropdb` | Remove an existing database. |
| `dropuser` | Delete an existing role. |
| `ecpg` | Preprocess C source files with embedded SQL. |
| `pg_amcheck` | Check indexes and heap data for corruption. |
| `pg_basebackup` | Take an online base backup over the replication protocol. |
| `pgbench` | Run benchmark tests on an IvorySQL instance. |
| `pg_config` | Print build-time configuration details. |
| `pg_dump` | Back up a single database. |
| `pg_dumpall` | Back up every database plus global objects. |
| `pg_isready` | Check whether a server is accepting connections. |
| `pg_receivewal` | Stream WAL segments from a running cluster. |
| `pg_recvlogical` | Manage logical-replication slots and stream decoded changes. |
| `pg_restore` | Restore archives produced by pg_dump. |
| `pg_verifybackup` | Validate a base backup against its backup manifest. |
| `psql` | Interactive terminal for running SQL. |
| `reindexdb` | Rebuild indexes for a database, schema, or table. |
| `vacuumdb` | Run VACUUM and ANALYZE on selected databases. |

### Server Applications

| Tool | Description |
|------|-------------|
| `initdb` | Initialize a new database cluster. |
| `pg_archivecleanup` | Clean archived WAL files no longer needed. |
| `pg_checksums` | Enable, disable, or verify data checksums. |
| `pg_controldata` | Display cluster control information from pg_control. |
| `pg_ctl` | Initialize, start, stop, or restart the server. |
| `pg_resetwal` | Reset WAL and control fields after corruption. |
| `pg_rewind` | Sync a diverged cluster with another timeline. |
| `pg_test_fsync` | Benchmark wal_sync_method options. |
| `pg_test_timing` | Measure timer resolution and detect backward time jumps. |
| `pg_upgrade` | Upgrade between major releases using existing data files. |
| `pg_waldump` | Read and print WAL records for debugging. |
| `postgres` | The IvorySQL server executable. |

## Client Applications

### clusterdb

Recluster tables in a database by invoking the SQL `CLUSTER` command.

**Synopsis**

```bash
clusterdb [connection-option...] [--verbose | -v] [--table | -t table] ... [dbname]
clusterdb [connection-option...] [--verbose | -v] --all | -a
```

**Key Options**

| Option | Description |
|--------|-------------|
| `-a`, `--all` | Cluster all databases. |
| `-d dbname`, `--dbname=dbname` | Database to cluster. Defaults to `PGDATABASE` or the connection user name. |
| `-e`, `--echo` | Echo the commands sent to the server. |
| `-q`, `--quiet` | Suppress progress messages. |
| `-t table`, `--table=table` | Cluster only the specified table. Repeat for multiple tables. |
| `-v`, `--verbose` | Print detailed processing information. |
| `--maintenance-db=dbname` | Database used to discover which databases to cluster when `--all` is used. |

**Environment**

`PGDATABASE`, `PGHOST`, `PGPORT`, `PGUSER` -- default connection parameters. `PG_COLOR` controls color output (`always`, `auto`, `never`).

**Examples**

```bash
# Cluster the database "test"
$ clusterdb test

# Cluster a single table "foo" in database "xyzzy"
$ clusterdb --table=foo xyzzy
```

### createdb

Create a new IvorySQL database, wrapping `CREATE DATABASE`.

**Synopsis**

```bash
createdb [connection-option...] [option...] [dbname [description]]
```

**Key Options**

| Option | Description |
|--------|-------------|
| `dbname` | Name of the database to create (must be unique in the cluster). |
| `description` | Comment for the new database. |
| `-D tablespace`, `--tablespace=tablespace` | Default tablespace for the database. |
| `-E encoding`, `--encoding=encoding` | Character encoding for the database. |
| `-l locale`, `--locale=locale` | Locale setting (equivalent to `--lc-collate` and `--lc-ctype`). |
| `--locale-provider=\{libc|icu\}` | Locale provider for default collation. |
| `-O owner`, `--owner=owner` | User who will own the new database. |
| `-T template`, `--template=template` | Template database to build from. |

**Examples**

```bash
# Create database "demo" on the default server
$ createdb demo

# Create database "demo" on a specific server with template0
$ createdb -p 5000 -h eden -T template0 -e demo
CREATE DATABASE demo TEMPLATE template0;
```

### createuser

Create a new IvorySQL role, wrapping `CREATE ROLE`.

**Synopsis**

```bash
createuser [connection-option...] [option...] [username]
```

**Key Options**

| Option | Description |
|--------|-------------|
| `-c number`, `--connection-limit=number` | Maximum connections for the new user. |
| `-d`, `--createdb` | Allow the new user to create databases. |
| `-e`, `--echo` | Echo generated commands. |
| `-g role`, `--role=role` | Add the new role as a member of the specified role. |
| `--interactive` | Prompt for user name and options not specified on the command line. |
| `-l`, `--login` | Allow the new user to log in (default). |
| `-P`, `--pwprompt` | Prompt for the new user's password. |
| `-r`, `--createrole` | Allow the new user to create roles. |
| `-s`, `--superuser` | Make the new user a superuser. |
| `--replication` | Grant the `REPLICATION` privilege. |

**Examples**

```bash
# Create a user "joe"
$ createuser joe

# Create with interactive prompts
$ createuser --interactive joe
Shall the new role be a superuser? (y/n) n
Shall the new role be allowed to create databases? (y/n) n

# Create as superuser with a password
$ createuser -P -s -e joe
Enter password for new role: xyzzy
CREATE ROLE joe PASSWORD 'md5...' SUPERUSER CREATEDB CREATEROLE INHERIT LOGIN;
```

### ecpg

Embedded SQL C preprocessor. Translates C source files with embedded SQL statements into standard C code that uses libpq.

**Synopsis**

```bash
ecpg [option...] file...
```

**Key Options**

| Option | Description |
|--------|-------------|
| `-c` | Automatically generate C code from SQL code (for `EXEC SQL TYPE`). |
| `-C mode` | Set compatibility mode: `INFORMIX`, `INFORMIX_SE`, or `ORACLE`. |
| `-I directory` | Add an include directory for `EXEC SQL INCLUDE`. |
| `-o filename` | Write output to the specified file (use `-` for stdout). |
| `-t` | Enable autocommit of transactions. |
| `-v` | Print additional version and include path information. |

**Examples**

```bash
ecpg prog1.pgc
cc -I/usr/local/pgsql/include -c prog1.c
cc -o prog1 prog1.o -L/usr/local/pgsql/lib -lecpg
```

### pg_basebackup

Take a base backup of an IvorySQL cluster over the replication protocol. Useful for PITR or standby seeding.

**Synopsis**

```bash
pg_basebackup [option...]
```

**Key Options**

| Option | Description |
|--------|-------------|
| `-D directory`, `--pgdata=directory` | Target directory for the backup (required). |
| `-F format`, `--format=format` | Output format: `plain` (default) or `tar`. |
| `-R`, `--write-recovery-conf` | Create `standby.signal` and append connection settings. |
| `-X method`, `--wal-method=method` | Include WAL: `none`, `fetch`, or `stream` (default). |
| `-Z level`, `--compress=level` | Enable compression (gzip, lz4, zstd). |
| `-c \{fast|spread\}`, `--checkpoint=\{fast|spread\}` | Checkpoint mode. |
| `-P`, `--progress` | Enable progress reporting. |
| `-T olddir=newdir`, `--tablespace-mapping` | Relocate tablespaces during backup. |

**Examples**

```bash
# Backup from a remote server
$ pg_basebackup -h mydbserver -D /usr/local/pgsql/data

# Compressed tar backup with progress
$ pg_basebackup -D backup -Ft -z -P

# Relocate a tablespace during backup
$ pg_basebackup -D backup/data -T /opt/ts=$(pwd)/backup/ts
```

### pgbench

Run benchmark tests (TPC-B style by default) on an IvorySQL instance.

**Synopsis**

```bash
pgbench -i [option...] [dbname]    # Initialize mode
pgbench [option...] [dbname]       # Benchmark mode
```

<Callout type="warn">

`pgbench -i` creates four tables (`pgbench_accounts`, `pgbench_branches`, `pgbench_history`, `pgbench_tellers`), destroying any existing tables of these names. Use a separate database to avoid data loss.

</Callout>

At the default scale factor of 1, the initial tables contain:

```
table                   # of rows
---------------------------------
pgbench_branches        1
pgbench_tellers         10
pgbench_accounts        100000
pgbench_history         0
```

**Initialization Options**

| Option | Description |
|--------|-------------|
| `-i`, `--initialize` | Invoke initialization mode. |
| `-s factor`, `--scale=factor` | Multiply rows by the scale factor. |
| `-F fillfactor`, `--fillfactor=fillfactor` | Set fillfactor for benchmark tables. |
| `--foreign-keys` | Create foreign key constraints between standard tables. |
| `--partition-method=NAME` | Partition `pgbench_accounts` by `range` or `hash`. |
| `--partitions=NUM` | Number of partitions for the accounts table. |

**Benchmarking Options**

| Option | Description |
|--------|-------------|
| `-c clients`, `--client=clients` | Number of concurrent client sessions. |
| `-t transactions`, `--transactions=transactions` | Transactions per client. |
| `-T seconds`, `--time=seconds` | Run for this many seconds instead of fixed transactions. |
| `-j threads`, `--jobs=threads` | Number of worker threads. |
| `-R rate`, `--rate=rate` | Target transactions per second (throttled mode). |
| `-L limit`, `--latency-limit=limit` | Report transactions exceeding this latency (ms). |
| `-b scriptname`, `--builtin=scriptname` | Built-in scripts: `tpcb-like`, `simple-update`, `select-only`. |
| `-f filename`, `--file=filename` | Run a custom transaction script. |
| `-P sec`, `--progress=sec` | Progress report interval in seconds. |
| `-r`, `--report-per-command` | Report per-command statistics after benchmark. |

**Exit Status**

`0` -- success. `1` -- static problems (invalid options, connection failure). `2` -- runtime errors (database errors, script problems).

### pg_dump

Back up a single IvorySQL database into a script file or archive.

**Synopsis**

```bash
pg_dump [connection-option...] [option...] [dbname]
```

**Key Options**

| Option | Description |
|--------|-------------|
| `-a`, `--data-only` | Dump only data, not schema. |
| `-c`, `--clean` | Include commands to drop objects before recreating. |
| `-C`, `--create` | Include commands to create the database. |
| `-e pattern`, `--extension=pattern` | Dump only matching extensions. |
| `-f filename`, `--file=filename` | Output file. |
| `-F format`, `--format=format` | Output format: `plain` (default), `custom`, `directory`, or `tar`. |
| `-j njobs`, `--jobs=njobs` | Use parallel dump (directory format only). |
| `-n schema`, `--schema=schema` | Dump only the specified schema. |
| `-O`, `--no-owner` | Skip ownership commands in output. |
| `-s`, `--schema-only` | Dump only schema, not data. |
| `-t table`, `--table=table` | Dump only the specified table. |
| `-Z level`, `--compress=level` | Compression level (0-9). |

**Examples**

```bash
# Dump a database to a plain SQL file
$ pg_dump mydb > mydb_backup.sql

# Dump in custom format with compression
$ pg_dump -Fc -Z 6 mydb > mydb.dump

# Dump a single table
$ pg_dump -t my_table mydb > table_backup.sql
```

### psql

The IvorySQL interactive terminal. Runs SQL interactively or from scripts.

**Synopsis**

```bash
psql [option...] [dbname [username]]
```

**Key Options**

| Option | Description |
|--------|-------------|
| `-c command`, `--command=command` | Execute a single command and exit. |
| `-d dbname`, `--dbname=dbname` | Database to connect to (can be a connection string). |
| `-f filename`, `--file=filename` | Execute commands from a file. |
| `-l`, `--list` | List all databases and exit. |
| `-p port`, `--port=port` | Port to connect to. |
| `-U username`, `--username=username` | User to connect as. |
| `-q`, `--quiet` | Suppress welcome and informational messages. |
| `-1`, `--single-transaction` | Wrap all `-c`/`-f` commands in a single transaction. |
| `-X`, `--no-psqlrc` | Skip startup configuration files. |
| `-e`, `--echo-queries` | Echo SQL commands sent to the server. |
| `-E`, `--echo-hidden` | Echo queries generated by `\d` commands. |
| `-H`, `--html` | HTML output mode. |
| `--csv` | CSV output mode. |

**Exit Status**

`0` -- finished normally. `1` -- fatal error. `2` -- connection lost (non-interactive). `3` -- script error with `ON_ERROR_STOP` set.

### pg_ctl

Initialize, start, stop, restart, or query the status of the IvorySQL server.

**Synopsis**

```bash
pg_ctl init[db] [-D datadir] [-s] [-o initdb-options]
pg_ctl start [-D datadir] [-l filename] [-W] [-t seconds] [-s] [-o options]
pg_ctl stop [-D datadir] [-m smart|fast|immediate] [-W] [-t seconds] [-s]
pg_ctl restart [-D datadir] [-m smart|fast|immediate] [-W] [-t seconds] [-s] [-o options]
pg_ctl reload [-D datadir] [-s]
pg_ctl status [-D datadir]
pg_ctl promote [-D datadir] [-W] [-t seconds] [-s]
```

**Key Options**

| Option | Description |
|--------|-------------|
| `-D datadir`, `--pgdata=datadir` | Data directory location. |
| `-l filename`, `--log=filename` | Append server log output to file. |
| `-m mode`, `--mode=mode` | Shutdown mode: `smart`, `fast` (default), or `immediate`. |
| `-o options`, `--options=options` | Options passed to `postgres`. |
| `-w`, `--wait` | Wait for the operation to complete (default for start/stop/restart). |
| `-W`, `--no-wait` | Do not wait for completion. |
| `-c`, `--core-files` | Allow core dumps on crash. |
| `-s`, `--silent` | Print only errors. |

**Shutdown Modes**

- **smart** -- Wait for all clients to disconnect before shutting down.
- **fast** -- Force disconnect all clients and shut down immediately (default).
- **immediate** -- Abort all server processes immediately; recovery will be required on next start.

**Examples**

```bash
# Start the server
$ pg_ctl start

# Start with custom options
$ pg_ctl -o "-F -p 5433" start

# Stop with fast shutdown
$ pg_ctl stop -m fast

# Show status
$ pg_ctl status
pg_ctl: server is running (PID: 13718)
/usr/local/pgsql/bin/postgres "-D" "/usr/local/pgsql/data" "-p" "5433"
```

### pg_upgrade

Upgrade between major IvorySQL releases by reusing existing data files.

**Synopsis**

```bash
pg_upgrade -b oldbindir [-B newbindir] -d oldconfigdir -D newconfigdir [option...]
```

**Key Options**

| Option | Description |
|--------|-------------|
| `-b bindir`, `--old-bindir=bindir` | Old IvorySQL executable directory. |
| `-B bindir`, `--new-bindir=bindir` | New IvorySQL executable directory. |
| `-d datadir`, `--old-datadir=datadir` | Old cluster data directory. |
| `-D datadir`, `--new-datadir=datadir` | New cluster data directory. |
| `-c`, `--check` | Check compatibility only without making changes. |
| `-k`, `--link` | Use hard links instead of copying (faster). |
| `--clone` | Use file cloning (reflinks) -- near-instantaneous. |
| `-j njobs`, `--jobs=njobs` | Parallel processing for schema dump/restore. |
| `-U username`, `--username=username` | Cluster superuser name. |

**Typical Upgrade Steps**

1. Build and install the new IvorySQL version.
2. Initialize the new cluster with `initdb`.
3. Install extension shared object files matching the new version.
4. Stop both servers.
5. Run `pg_upgrade --check` to verify compatibility.
6. Run `pg_upgrade` with appropriate options.
7. Start the new server.
8. Run `ANALYZE` to regenerate optimizer statistics.
9. Delete the old cluster when satisfied.

**Example**

```bash
# Check mode (old server can still be running)
$ pg_upgrade -b /opt/IvorySQL/4/bin -B /opt/IvorySQL/5/bin \
  -d /opt/IvorySQL/4/data -D /opt/IvorySQL/5/data --check

# Upgrade with link mode
$ pg_upgrade -b /opt/IvorySQL/4/bin -B /opt/IvorySQL/5/bin \
  -d /opt/IvorySQL/4/data -D /opt/IvorySQL/5/data --link
```

### postgres

The IvorySQL server executable. Accepts client connections and forks per-session backend processes.

**Synopsis**

```bash
postgres [option...]
```

**General Purpose Options**

| Option | Description |
|--------|-------------|
| `-D datadir` | Data directory location. |
| `-c name=value` | Set a run-time parameter. |
| `-p port` | Port to listen on (default: 5432). |
| `-h hostname` | Listen address for TCP/IP connections. |
| `-k directory` | Unix-domain socket directory. |
| `-N max-connections` | Maximum client connections. |
| `-B nbuffers` | Shared buffers (auto-set by initdb). |
| `-F` | Disable fsync (performance risk). |
| `-l` | Enable SSL connections. |

**Example**

```bash
# Start with custom port
$ postgres -p 1234

# Set parameters
$ postgres -c shared_buffers=256MB -c max_connections=200
```

<Callout type="info">

For normal operation, use `pg_ctl` to start and stop the server safely rather than invoking `postgres` directly.

</Callout>
