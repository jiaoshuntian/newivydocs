# Migrate from Oracle

IvorySQL is purpose-built for Oracle-to-PostgreSQL migrations. It understands Oracle SQL dialects, PL/SQL syntax, Oracle data types, and built-in functions natively — so you can migrate incrementally, test compatibility before you cut over, and avoid the big-bang rewrite.

## Migration goals

A successful migration to IvorySQL means:

- **No data loss** — every row, every column, every constraint survives intact
- **Full functional compatibility** — Oracle SQL and PL/SQL run without modification where supported
- **Upstream compatibility** — IvorySQL stays in sync with the latest PostgreSQL; migrated workloads benefit from every PostgreSQL improvement
- **Performance parity** — production query response times match or exceed Oracle

## Recommended migration path

<Steps>

### Assess compatibility

Review the [Oracle Compatibility](/docs/v5/oracle-compatibility) section to identify which Oracle SQL features, data types, and PL/SQL constructs are supported natively in IvorySQL.

### Configure IvorySQL

[Enable Oracle compatibility mode](/docs/v5/migration/compatibility-mode) on your target cluster, and [configure dual-port access](/docs/v5/migration/ports-ip) so Oracle client connection strings work without change.

### Migrate schemas and data

Run [Ora2Pg](/docs/v5/migration/migrate-ora2pg) to export your Oracle schema and data. Ora2Pg generates IvorySQL-compatible SQL scripts for tables, sequences, indexes, views, stored procedures, and data.

### Validate and cut over

Run your test suite against the migrated database. Compare row counts and spot-check data. Validate stored procedures and triggers. When the results match, update your application connection string and cut over.

</Steps>
