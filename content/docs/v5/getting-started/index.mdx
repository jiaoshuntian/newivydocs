# What is IvorySQL?

IvorySQL is an advanced, open-source PostgreSQL distribution with full Oracle compatibility. It is built and maintained by [Highgo Software](https://www.highgo.com) and is Apache-2.0 licensed ‚Äî free to use, modify, and distribute forever.

The core promise: **run your Oracle workloads on PostgreSQL without rewriting your application**. IvorySQL speaks both dialects at once. You can connect an Oracle application on port 1521 and a native PostgreSQL tool on port 5432 ‚Äî against the same running instance.

<Callout type="info">
IvorySQL tracks the latest PostgreSQL major release. IvorySQL 5.x is built on PostgreSQL 18 and is a direct, drop-in replacement for PostgreSQL 18.
</Callout>

## The dual-mode concept

IvorySQL introduces a GUC parameter called `ivorysql.compatible_mode` with two values: `oracle` and `pg`.

When you initialize a data directory you choose its baseline mode with the `-m` flag:

- **`-m oracle`** (the default) ‚Äî Oracle compatibility is active at startup. Every connection begins in Oracle mode. You can switch individual sessions to `pg` mode with `SET ivorysql.compatible_mode TO pg`.
- **`-m pg`** ‚Äî The cluster behaves like standard PostgreSQL. Oracle features are not available and the mode cannot be changed at runtime.

This means a single IvorySQL cluster can serve Oracle-style applications and PostgreSQL-native applications at the same time, simply by choosing which port to connect on.

## What Oracle features are included?

Oracle compatibility is delivered through a kernel-bound plugin called `ivorysql_ora`, which ships with the database and requires no manual installation.

In Oracle mode, the following work out of the box:

- **PL/iSQL** ‚Äî Oracle PL/SQL syntax for functions, procedures, packages, anonymous blocks, and nested subfunctions
- **Data types** ‚Äî `NUMBER`, `VARCHAR2`, `DATE`, `CLOB`, `BLOB` and more
- **Built-in functions** ‚Äî `SYSDATE`, `TO_CHAR`, `TO_DATE`, `NVL`, `DECODE`, `TRUNC`, `SYS_GUID`, and others
- **SQL syntax** ‚Äî `SELECT ‚Ä¶ FROM dual`, `ROWNUM`, Oracle `LIKE` behavior, `CONNECT BY` hierarchical queries
- **Schema objects** ‚Äî Oracle-style sequences (`seq.NEXTVAL`), packages, invisible columns, force views
- **Type attributes** ‚Äî `%TYPE`, `%ROWTYPE`, OUT parameters, `CALL INTO`
- **Character set** ‚Äî GB18030 and NLS parameter support
- **XML functions** ‚Äî Oracle-compatible XML built-ins

## Key design decisions

**Always compatible with upstream PostgreSQL.** IvorySQL never breaks PostgreSQL compatibility. Every Oracle feature is additive ‚Äî you lose nothing by running IvorySQL instead of PostgreSQL.

**Plugin-based extensibility.** Oracle-compatible features are delivered via `ivorysql_ora`, a plugin bound to the kernel. New compatibility features are added through this plugin without changing the core engine, making upgrades predictable.

**Dual-port architecture.** IvorySQL listens on port 1521 (Oracle default) and port 5432 (PostgreSQL default) simultaneously. The port you connect on sets the default compatibility mode for that session, so existing Oracle connection strings work without reconfiguration.

**Open source and community-driven.** The full source code ‚Äî including all Oracle-compatible features ‚Äî is published on [GitHub](https://github.com/IvorySQL/IvorySQL) under the Apache 2.0 license. The feature roadmap is shaped by the community.

## Core application scenarios

- **Enterprise systems** ‚Äî ERP, financial platforms, and transaction-heavy workloads that need Oracle SQL and PL/SQL with PostgreSQL reliability and cost
- **Location-based services** ‚Äî PostGIS integration gives you geographic queries in SQL
- **Data warehousing** ‚Äî Rich data types and powerful aggregations for analytics workloads
- **Oracle migration** ‚Äî Migrate Oracle schemas and applications incrementally, without a big-bang cutover
