# Release Notes

## IvorySQL 5.4

**Release Date:** June 17, 2026

IvorySQL 5.4 is built on PostgreSQL 18.4 and includes a variety of bug fixes and new features.

### New Features

- **PostgreSQL 18.4 kernel upgrade** — [PR #1344](https://github.com/IvorySQL/IvorySQL/pull/1344)
- **[IvorySQL Skills](https://github.com/IvorySQL/skills)** — AI assistant plugins for IvorySQL database expertise
- **[Online Experience](http://trial.ivorysql.org/)** — Browser-based interactive environment [PR #10](https://github.com/IvorySQL/ivorysql-wasm/pull/10)
- **Multi-platform packaging** — X86, ARM, MIPS, LoongArch architecture support
- **Containerized deployment** — One-click deployment via Docker Compose, Podman, Swarm, Helm, and IvorySQL Operator
- **IvorySQL Cloud 5.4** — Fully managed graphical control plane
- **10 PostgreSQL extensions** — pg_cron, pgAudit, PostGIS, pgRouting, PGroonga, ddlx, pgsql-http, system_stats, plpgsql_check, pgvector
- **Enhanced EXTRACT function** — Additional syntax support [PR #1222](https://github.com/IvorySQL/IvorySQL/pull/1222)

### PostgreSQL 18.4 Upstream Fixes

1. Prevent unbounded recursion while processing startup packets
2. Fix integer overflows in memory-allocation calculations
3. Properly quote subscription names in `pg_createsubscriber`
4. Properly quote object names in logical replication origin checks
5. Reject over-length options in `ts_headline()`
6. Detect faulty input when restoring attribute MCV statistics

See [PostgreSQL 18.4 release notes](https://www.postgresql.org/docs/release/18.4/) for details.

## IvorySQL 5.3

**Release Date:** March 12, 2026

IvorySQL 5.3 is built on PostgreSQL 18.3 and includes a variety of bug fixes.

### Enhancements

**PostgreSQL 18.3 upstream fixes:**

1. Fix failure after replaying a multixid truncation record from WAL generated by an older minor version
2. Avoid incorrect complaint of invalid encoding when `substring()` is applied to "toasted" data
3. Fix oversight in the fix for CVE-2026-2007
4. Fix the volatility marking of `json_strip_nulls()` and `jsonb_strip_nulls()`
5. Fix computation of the set of potentially-nulling outer joins for the output of a `LATERAL UNION ALL` subquery

See [PostgreSQL 18.3 release notes](https://www.postgresql.org/docs/release/18.3/) for details.

**PostgreSQL 18.2 upstream fixes:**

1. Guard against unexpected dimensions of `oidvector`/`int2vector`
2. Harden selectivity estimators against being attached to operators that accept unexpected data types
3. Fix buffer overrun in `contrib/pgcrypto`'s PGP decryption functions
4. Fix inadequate validation of multibyte character lengths
5. Harden `contrib/pg_trgm` against changes in string lowercasing behavior

See [PostgreSQL 18.2 release notes](https://www.postgresql.org/docs/release/18.2/) for details.

## Prior Versions

<Callout type="info">
Full release notes for all versions are available on [GitHub Releases](https://github.com/IvorySQL/IvorySQL/releases).
</Callout>

### IvorySQL 4.x (PostgreSQL 17)

IvorySQL 4.x is built on the PostgreSQL 17 kernel. It introduces improvements to logical replication, `MERGE` enhancements, and continues Oracle-compatibility expansion.

### IvorySQL 3.x (PostgreSQL 16)

IvorySQL 3.x is built on the PostgreSQL 16 kernel. Notable additions include parallel `pg_dump`, logical replication from standbys, and enhanced `COPY FROM` error handling.

### IvorySQL 2.x (PostgreSQL 15)

IvorySQL 2.x is built on the PostgreSQL 15 kernel. Key highlights include `MERGE` command support, `pg_basebackup` compression, and improved logical-replication conflict detection.

### IvorySQL 1.x (PostgreSQL 14)

IvorySQL 1.x is the initial release series, built on the PostgreSQL 14 kernel. It introduced the Oracle-compatible parser, `PL/iSQL` procedural language, dual compatibility mode (`ivorysql.compatible_mode`), and core Oracle-compatible data types and functions.
