# Core Concepts

IvorySQL adds Oracle compatibility to PostgreSQL without forking the engine. This page explains the key design decisions and helps you decide how to configure it for your use case.

## Which mode should I use?

<Callout type="tip">
**Use Oracle mode if** you are migrating from Oracle, running PL/SQL code, or need Oracle built-in functions like `SYSDATE`, `NVL`, `TO_CHAR`, or `ROWNUM`.

**Use PG mode if** you are building a new application from scratch, need 100% PostgreSQL syntax compatibility, or are integrating with tools that expect standard PostgreSQL behavior.

You can switch modes at runtime with `SET ivorysql.compatible_mode TO pg` â€” as long as the cluster was initialized in Oracle mode.
</Callout>

## Dual-mode architecture

When you create a cluster, you choose its default compatibility mode using the `-m` flag on `initdb`:

```bash
# Oracle-compatible mode (this is the default if -m is omitted)
initdb -D /usr/ivory-5/data -m oracle

# PostgreSQL mode
initdb -D /usr/ivory-5/data -m pg
```

The mode you choose at initialization sets the baseline. In Oracle mode, every connection starts with Oracle compatibility active. In PG mode, Oracle features are never available â€” the mode cannot be switched later.

## Dual-port design

IvorySQL listens on two ports. The port you connect on determines the default compatibility mode for that session:

| Port | Default mode | Use for |
|------|-------------|---------|
| **1521** | Oracle | Oracle applications, migrations, PL/SQL |
| **5432** | PostgreSQL | PG-native tools, drivers, ORMs |

If you connect on 5432 and need Oracle mode for a specific session, run:

```sql
SET ivorysql.compatible_mode TO oracle;
```

## What Oracle features work out of the box?

When Oracle mode is active, these capabilities are available without any configuration:

**Procedural language**
- PL/iSQL (Oracle PL/SQL syntax for functions, procedures, packages)
- Anonymous blocks (`BEGIN â€¦ END; /`)
- Nested subfunctions, OUT parameters, `%TYPE` / `%ROWTYPE`

**Data types and functions**
- `NUMBER`, `VARCHAR2`, `DATE`, `CLOB`, `BLOB`
- `SYSDATE`, `TO_CHAR`, `TO_DATE`, `NVL`, `DECODE`, `TRUNC`
- `SYS_GUID()` for UUID generation

**SQL syntax**
- `SELECT â€¦ FROM dual`
- `ROWNUM` pseudo-column
- Oracle `LIKE` wildcard behavior
- `ROWID` pseudo-column
- Empty string treated as `NULL`
- `CONNECT BY` hierarchical queries

**Schema objects**
- Oracle-style sequences (`seq.NEXTVAL`, `seq.CURRVAL`)
- Packages (`CREATE PACKAGE â€¦ END;`)
- Invisible columns
- Force views

## The ivorysql_ora plugin

Oracle compatibility is implemented in a plugin called `ivorysql_ora`, which is bundled with the database. It provides the Oracle built-in functions, system views, data types, and GUC parameters. New compatibility features are delivered via this plugin as IvorySQL evolves.

You do not need to install or configure this plugin manually â€” it is active whenever Oracle mode is enabled.
