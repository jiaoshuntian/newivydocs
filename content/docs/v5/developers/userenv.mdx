# USERENV Implementation

IvorySQL provides compatibility with Oracle's built-in function `USERENV('parameter')`, which returns information about the current session.

<Callout type="info">
`USERENV` is a legacy function. IvorySQL recommends using the `SYS_CONTEXT` function with its built-in `USERENV` namespace for current functionality. See [SYS_CONTEXT](/docs/v5/developers/built-in-functions/sys-context) for details.
</Callout>

## Implementation Principle

`USERENV` parses function calls through Bison rules, checks parameter validity, and maps them to corresponding SQL functions. The parsing rules are implemented in `ora_gram.y`:

```c
USERENV '(' Sconst ')'
    {
        char *normalized_param = downcase_identifier($3, strlen($3), true, true);

        #define CHECK_AND_CALL(param, func_name) \
            if (strcmp(normalized_param, param) == 0) \
                $$ = (Node *) makeFuncCall(OracleSystemFuncName(func_name), NIL, COERCE_EXPLICIT_CALL, @1);

        CHECK_AND_CALL("client_info", "get_client_info")
        else CHECK_AND_CALL("entryid", "get_entryid")
        else CHECK_AND_CALL("terminal", "get_terminal")
        else CHECK_AND_CALL("isdba", "get_isdba")
        else CHECK_AND_CALL("lang", "get_lang")
        else CHECK_AND_CALL("language", "get_language")
        else CHECK_AND_CALL("sessionid", "get_sessionid")
        else CHECK_AND_CALL("sid", "get_sid")
        else
            ereport(ERROR,
                    (errcode(ERRCODE_FEATURE_NOT_SUPPORTED),
                     errmsg("invalid USERENV parameter: \"%s\".", $3)));
        #undef CHECK_AND_CALL
    }
```

The pattern is a `CHECK_AND_CALL` macro that:

1. Normalizes the input parameter to lowercase.
2. Compares it against known parameter names.
3. If matched, constructs a `FuncCall` AST node calling the corresponding internal function (e.g., `sys.get_sid()`).
4. If no match, raises an error.

## Underlying SQL Functions

The specific functionality is implemented in `builtin_functions--1.0.sql`. For example:

```sql
CREATE OR REPLACE FUNCTION sys.get_language()
RETURNS varchar2
AS $$
    SELECT (regexp_split_to_array(current_setting('lc_monetary'), '\\.'))[1]||'.'||pg_client_encoding();
$$ LANGUAGE sql STRICT;
```

Each `USERENV` parameter maps to a dedicated function in the `sys` schema. These functions query PostgreSQL system catalogs, GUC parameters, or session state to return the appropriate value.

## Supported Parameters

| Parameter | Return Value |
|---|---|
| `sid` | The user ID of the current session. |
| `sessionid` | Returns the audit session identifier. You cannot use this attribute in distributed SQL statements. |
| `language` | Returns the language, territory, and character set of the current database session in the form `language_territory.characterset`. |
| `lang` | Returns the ISO abbreviation of the language name. A shorter form than `LANGUAGE`. |
| `isdba` | Returns `TRUE` if the user has been authenticated as an administrator or has administrator privileges through the operating system or password file; otherwise returns `FALSE`. |
| `client_info` | Returns up to 64 bytes of user session information that can be stored by an application using `DBMS_APPLICATION_INFO`. |
| `entryid` | The current audit entry number. The audit entry ID sequence is shared between fine-grained audit records and regular audit records. You cannot use this attribute in distributed SQL statements. |
| `terminal` | The operating system identifier for the client of the current session. In distributed SQL statements, this returns the identifier for the local session. |
