# Empty String to NULL Implementation

In Oracle databases, empty strings (`''`) are treated as `NULL` values. To maintain compatibility with this Oracle behavior, IvorySQL provides the empty string to NULL conversion functionality. When enabled, empty strings in SQL statements are automatically converted to `NULL` values.

## GUC Parameter

The parameter `ivorysql.enable_emptystring_to_NULL` corresponds to the GUC variable `enable_emptystring_to_NULL`.

## Lexical Analysis

In `ora_scan.l`, the conversion happens during lexical analysis:

```c
case xe:
    yylval->str = litbufdup(yyscanner);
    if (strcmp(yylval->str, "") == 0 &&
        ORA_PARSER == compatible_db &&
        enable_emptystring_to_NULL)
    {
        return NULL_P;
    }
    return SCONST;
```

Here `xe` represents extended quoted strings (strings enclosed in quotes that support backslash escape sequences). The logic:

1. During lexical analysis, if an empty string is encountered.
2. AND the Oracle parser is active (`ORA_PARSER == compatible_db`).
3. AND the GUC parameter `enable_emptystring_to_NULL` is enabled.
4. Then return `NULL_P` instead of `SCONST`.

## Grammar Analysis

In `ora_gram.y`, the statement `insert into abc values('');` is parsed through the grammar chain:

```
values_clause: VALUES '(' expr_list ')'
expr_list:     a_expr
a_expr:        c_expr
c_expr:        AexprConst
AexprConst:    Sconst { $$ = makeStringConst($1, @1); }
           |   NULL_P { $$ = makeNullAConst(@1); }
```

The code constructs corresponding AST nodes to handle `NULL_P` or `SCONST` returned from the lexical analysis. When the lexical analyzer returns `NULL_P` (due to empty string conversion), the grammar constructs a null constant node via `makeNullAConst()` instead of a string constant node.

This means the conversion happens entirely at the lexical analysis stage — the grammar layer is agnostic to whether the `NULL_P` token originated from an explicit `NULL` keyword or from an empty string conversion.
