## NLS Parameter Reference

The following GUC parameters are part of IvorySQL's NLS compatibility layer. They are defined in `ivy_guc.c` and resolved at parse or execution time depending on the feature.

| Parameter | Description |
|-----------|-------------|
| `ivorysql.datetime_ignore_nls_mask` | Bitmask controlling which date/time types bypass NLS formatting. Default: `0`. |
| `nls_length_semantics` | Whether `CHAR`/`VARCHAR`/`VARCHAR2` length modifiers count bytes or characters. Default: `BYTE`. |
| `nls_date_format` | Default format for `DATE` input/output. Default: `YYYY-MM-DD`. |
| `nls_timestamp_format` | Default format for `TIMESTAMP`. Default: `YYYY-MM-DD HH24:MI:SS.FF6`. |
| `nls_timestamp_tz_format` | Default format for `TIMESTAMP WITH TIME ZONE`. |
| `nls_territory` | Default region for currency and locale. Supported: `AMERICA`, `CHINA`. |
| `nls_iso_currency` | ISO currency symbol tied to the territory. |
| `nls_currency` | Local currency symbol, used as the `L` placeholder in numeric formats. |

<Callout type="info">
  Setting `nls_date_format`, `nls_timestamp_format`, or `nls_timestamp_tz_format` to `"pg"` disables NLS-specific formatting and reverts to PostgreSQL default behavior.
</Callout>
