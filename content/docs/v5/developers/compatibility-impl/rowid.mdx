# RowID Implementation

IvorySQL provides Oracle-compatible RowID functionality. RowID is a pseudo-column automatically generated by the database when a table is created, returning the address of each row in the database.

RowID has the following characteristics:

1. Logically identifies each row with a unique value.
2. Allows quick querying and modification of other columns in the table via ROWID, but cannot be inserted or modified itself.
3. Users can control whether this feature is enabled.

## Implementation Principles

In IvorySQL, the system column `ctid` represents the physical location of a data row in a table (block number and row index). While `ctid` allows for quick lookup, its value may change during UPDATE or VACUUM FULL, making it unsuitable as a long-term row identifier.

IvorySQL chose a composite type consisting of the table's OID and a sequence value as the RowID value. If RowID is enabled, a sequence named `table-id_rowid_seq` is created simultaneously with the table.

### Storage Modifications

In `heap_form_tuple`, the length of `HeapTupleHeaderData` is increased by 8 bytes, and `td->t_infomask = HEAP_HASROWID` is set to indicate the presence of RowID.

### Sequence Creation

When RowID is enabled (via GUC parameter, `WITH ROWID` option in `CREATE TABLE`, or `ALTER TABLE ... SET WITH ROWID`), a sequence creation command is added:

```c
seqstmt = makeNode(CreateSeqStmt);
seqstmt->with_rowid = true;
seqstmt->sequence = makeRangeVar(snamespace, sname, -1);
seqstmt->options = lcons(makeDefElem("as",
                     (Node *) makeTypeNameFromOid(INT8OID, -1), -1),
                 seqstmt->options);
seqstmt->options = lcons(makeDefElem("nocache", NULL, -1),
                 seqstmt->options);
```

### System Column Registration

The RowID column is implemented as a system attribute column by adding a new entry in `heap.c`:

```c
static const FormData_pg_attribute a7 = {
    .attname = {"rowid"},
    .atttypid = ROWIDOID,
    .attlen = -1,
    .attnum = RowIdAttributeNumber,
    .attcacheoff = -1,
    .atttypmod = -1,
    .attbyval = false,
    .attalign = TYPALIGN_SHORT,
    .attstorage = TYPSTORAGE_PLAIN,
    .attnotnull = true,
    .attislocal = true,
};
```

### pg_class Extension

A boolean field `relhasrowid` was added to the `pg_class` system table to indicate whether the `WITH ROWID` option was specified during table creation. This value is updated when `ALTER TABLE ... SET WITH ROWID` or `ALTER TABLE ... SET WITHOUT ROWID` is executed.

### Fast Query Performance

To enable fast querying via the RowID pseudo-column, a `UNIQUE` index is automatically created on the RowID column of the table by default.

### Sequence Value Assignment

In `heap_prepare_insert`, when RowID is enabled, `nextval` of the sequence is obtained and stored in the `HeapTupleHeader`:

```c
if (relation->rd_rel->relhasrowid) {
    seqnum = nextval_internal(relation->rd_rowdSeqid, true);
    HeapTupleSetRowId(tup, seqnum);
}
```
