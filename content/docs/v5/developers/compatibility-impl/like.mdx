# LIKE Operator Implementation

The LIKE syntax in Oracle and IvorySQL is identical. Their difference lies in the expression types: Oracle supports using the LIKE keyword with wildcards for fuzzy queries on columns of numeric, date, and string types. Native PostgreSQL only supports string types, not date or numeric types. IvorySQL achieves Oracle-compatible LIKE operator functionality by extending data type support and operator overloading.

## Implementation Principle

In PostgreSQL, the fundamental string type is `text`, so LIKE is `text`-based. Other PostgreSQL types implicitly convert to `text`, enabling automatic conversion without creating separate operators. In IvorySQL, the Oracle-compatible string type is `varchar2`. Therefore, a LIKE operator for `varchar2` is created, and other Oracle types utilize the LIKE operator by implicitly converting to `varchar2` without requiring additional operator creation.

IvorySQL established implicit conversions from certain data types (such as `integer`, `float8`, `float4`) to `varchar2`, but not directly to `text`.

To achieve LIKE operator compatibility, IvorySQL chose to implement a base LIKE operator for `varchar2`. Since IvorySQL already supports implicit conversions from `float8`, `integer`, `number`, etc., to `varchar2`, these data types can share the same operator. Only a single LIKE operator for `varchar2` needs to be created.

## Type Conversion

You can check existing implicit type conversions with:

```sql
-- Check existing implicit conversion paths (where 9503 is the OID of varchar2)
SELECT t1.typname AS source_type, t2.typname AS target_type
FROM pg_cast C
JOIN pg_type t1 ON C.castsource = t1.OID
JOIN pg_type t2 ON C.casttarget = t2.OID
WHERE C.casttarget = 9503;
```

All types requiring LIKE compatibility have implicit conversions defined in `pg_cast`, allowing them to be directly converted for fuzzy query operations.

## Core Function Implementation

```sql
CREATE OR REPLACE FUNCTION sys.varchar2like(varchar2, varchar2)
RETURNS bool AS $$
SELECT $1::text LIKE $2::text;
$$ LANGUAGE SQL IMMUTABLE STRICT;

CREATE OPERATOR ~~ (
  PROCEDURE = sys.varchar2like,
  LEFTARG   = varchar2,
  RIGHTARG  = varchar2
);
```

The implementation consists of two parts:

1. **`sys.varchar2like` function** — Accepts two `varchar2` parameters, casts them to PostgreSQL's native `text` type, and performs standard LIKE pattern matching. Returns a boolean.

2. **`~~` operator** — The internal PostgreSQL operator for LIKE. It uses `sys.varchar2like` as its implementation function with both operands typed as `varchar2`.

When users employ the `~~` operator with `varchar2` operands, it essentially invokes PostgreSQL's native LIKE functionality after type conversion, achieving semantic compatibility with Oracle's LIKE behavior.
