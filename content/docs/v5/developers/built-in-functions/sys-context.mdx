# SYS_CONTEXT Implementation

IvorySQL provides compatibility with the Oracle built-in function `SYS_CONTEXT('namespace', 'parameter' [, length])`, which returns the value of the parameter associated with the given context at the current moment. It can be used in both SQL and PL/SQL languages.

IvorySQL provides the following built-in namespaces:

- **USERENV** — Describes the current session.
- **SYS_SESSION_ROLES** — Indicates whether a specified role is currently enabled for the session.

## Implementation Architecture

The `SYS_CONTEXT` function dynamically queries system tables and PostgreSQL's built-in functions to ensure real-time results. It uses `SECURITY INVOKER` to ensure the function executes with the caller's permissions, avoiding privilege escalation issues.

The core implementation is a PL/iSQL function:

```sql
CREATE OR REPLACE FUNCTION sys.sys_context(a varchar2, b varchar2)
RETURNS varchar2 AS $$
DECLARE
    res varchar2;
BEGIN
    IF upper(a) = 'USERENV' THEN
      CASE upper(b)
          WHEN 'CURRENT_SCHEMA' THEN
            SELECT current_schema() INTO res;
          WHEN 'LANG' THEN
            SELECT sys.get_lang() INTO res;
          -- ... additional parameters
          ELSE
            RAISE EXCEPTION 'invalid USERENV parameter: %', b;
      END CASE;
    ELSIF upper(a) = 'SYS_SESSION_ROLES' THEN
        CASE upper(b)
            WHEN 'LOGIN' THEN
                SELECT CASE WHEN rolcanlogin = 't' THEN 'TRUE' ELSE 'FALSE' END
                INTO res FROM pg_roles WHERE oid = current_user::regrole::oid;
            -- ... additional parameters
            ELSE
                RAISE EXCEPTION 'invalid SYS_SESSION_ROLES parameter: %', b;
        END CASE;
    ELSE
      SELECT current_setting(a||'.'||b, true) INTO res;
    END IF;
    RETURN res;
END;
$$ LANGUAGE plisql SECURITY INVOKER;
```

The design uses a `CASE` chain inside a PL/iSQL block. Each parameter maps to either a PostgreSQL built-in function (like `current_schema()`) or an IvorySQL-provided function (like `sys.get_lang()`). For unrecognized namespaces, it falls back to `current_setting()` for custom GUC parameters.

## USERENV Parameters

The following parameters are supported in the `USERENV` namespace:

| Parameter | Return Value |
|---|---|
| `CURRENT_SCHEMA` | The name of the currently active default schema. May change during a session via `ALTER SESSION SET CURRENT_SCHEMA` or when entering definer's rights objects. |
| `CURRENT_SCHEMAID` | Identifier of the currently active default schema. |
| `SESSION_USER` | The name of the session user (the user who logged on). For enterprise users, returns the schema. |
| `SESSION_USERID` | The identifier of the session user. |
| `PROXY_USER` | Name of the database user who opened the current session on behalf of `SESSION_USER`. |
| `PROXY_USERID` | Identifier of the database user who opened the current session on behalf of `SESSION_USER`. |
| `CURRENT_USER` | The name of the database user whose privileges are currently active. May differ from `SESSION_USER` when definer's rights objects are active. |
| `CURRENT_USERID` | The identifier of the database user whose privileges are currently active. |
| `CURRENT_EDITION_NAME` | The name of the current edition. |
| `CLIENT_PROGRAM_NAME` | The name of the program used for the database session. |
| `IP_ADDRESS` | The IP address of the client. |
| `HOST` | Name of the host machine from which the client has connected. |
| `ISDBA` | Returns `TRUE` if the user has been authenticated as having DBA privileges. |
| `LANG` | The abbreviated name for the language. |
| `LANGUAGE` | The language, territory, and database character set in the form `language_territory.characterset`. |
| `NLS_DATE_FORMAT` | The date format for the session. |
| `PLATFORM_SLASH` | The slash character used as the file path delimiter for the platform. |
| `DB_NAME` | Name of the database as specified in the `DB_NAME` initialization parameter. |
| `SID` | The session ID. |
| `SESSIONID` | The auditing session identifier. |
| `CLIENT_INFO` | Up to 64 bytes of user session information set via `DBMS_APPLICATION_INFO`. |
| `ENTRYID` | The current audit entry number. |
| `TERMINAL` | The operating system identifier for the client of the current session. |

## SYS_SESSION_ROLES Parameters

| Parameter | Return Value |
|---|---|
| `DBA` | Returns `TRUE` if the current user is a database administrator. |
| `LOGIN` | Returns `TRUE` if the current user is a login role. |
| `CREATEROLE` | Returns `TRUE` if the current session's user has the privilege to create roles. |
| `CREATEDB` | Returns `TRUE` if the current session's user has the privilege to create databases. |
