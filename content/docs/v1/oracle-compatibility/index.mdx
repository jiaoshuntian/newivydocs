---
title: Oracle Compatibility
description: Overview of IvorySQL's Oracle compatibility features, including dual-mode operation, PL/iSQL, built-in data types, and GUC parameters.
---

IvorySQL is designed to be compatible with Oracle databases while maintaining full PostgreSQL compatibility. The Oracle compatibility layer allows you to run Oracle SQL syntax, use Oracle-style PL/SQL procedural code, and work with Oracle built-in data types and functions — all without leaving your PostgreSQL-based deployment.

The compatibility system is controlled by the `ivorysql.compatible_mode` GUC parameter and the Oracle-mode port (default: 1521). When you connect through port 1521, Oracle compatibility mode is active by default.

<Callout type="info">
  The `ivorysql.compatible_mode` parameter can be set to `oracle` or `pg` at the session level. Use `SET ivorysql.compatible_mode TO oracle;` to activate Oracle compatibility within an existing session connected to the standard port 5432.
</Callout>
